Section 3 – Getting Started with AWS

▼

Regions & Availability Zones

  • Region = Geographic area (e.g., eu-west-1)
  • AZ = One or more data centers within a Region
  • AZs are physically separate but connected via low-latency network

Exam Triggers

  • Multi-AZ → Protects against AZ failure
  • Multi-Region → Protects against Region failure
  • Most services are Region-scoped

Global vs Regional Services

Global Services (Memorize)

  • IAM
  • Route 53
  • CloudFront
  • WAF

Everything else → Usually Regional.

Shared Responsibility Model

AWS Responsible For:

  • Physical hardware
  • Data centers
  • Managed service infrastructure

Customer Responsible For:

  • IAM configuration
  • Security Groups
  • OS patching (EC2)
  • Application security
  • Data encryption configuration

Exam trigger: OS patching on EC2 → Customer responsibility.

Section 4 – IAM & AWS CLI

▼

IAM Users & Groups

  • Users → People
  • Groups → Collection of users
  • Groups cannot contain other groups
  • Users can belong to multiple groups

IAM Policies

Policy Structure (Must Know)

  • Version
  • Statement
  • Effect
  • Action
  • Resource

IAM Policy Evaluation Logic (MOST IMPORTANT)

Evaluation Order:

  1. Explicit Deny → ALWAYS wins
  2. Explicit Allow → Works if no Deny
  3. Implicit Deny → Default
Explicit Deny ↓ Explicit Allow ↓ Implicit Deny

If any policy says "Effect": "Deny" → Access denied.

IAM Roles

  • Used by AWS services (EC2, Lambda)
  • Provide temporary credentials
  • No need to store access keys

EC2 Access to S3

Correct solution: Attach IAM Role to EC2

Never store credentials on instance.

Instance Metadata

URL: 169.254.169.254

  • Provides temporary credentials
  • Used with IAM roles

Root Account

  • Enable MFA
  • Do NOT create access keys
  • Do NOT use daily
  • Use IAM users instead

Access Keys

  • Used for programmatic access
  • Rotate regularly
  • Never create for root

Section 5 – EC2 Fundamentals

▼

EC2 Instance Types

Categories:

  • General Purpose
  • Compute Optimized
  • Memory Optimized
  • Storage Optimized

Exam keyword: High CPU → Compute optimized.

In-memory DB → Memory optimized.

EC2 Lifecycle

States: Pending, Running, Stopping, Stopped, Terminated

Important:

  • Stop → EBS persists
  • Stop → Instance Store lost
  • Terminated → Cannot restart

Security Groups

  • Stateful firewall
  • Attached to ENI
  • Inbound + Outbound rules
  • If inbound allowed → return traffic auto allowed

SSH not working → Check inbound port 22.

Public vs Private EC2

Public EC2 requires:

  • Internet Gateway attached
  • Route table 0.0.0.0/0 → IGW
  • Public subnet
  • Public IP
  • Security Group allowing inbound

Missing one → No internet access.

Elastic IP

  • Static public IP
  • Survives stop/start
  • Region-scoped
  • Charged if unused
  • Limited per region

Purchasing Options

TypeUse Case
On-DemandShort-term
ReservedLong-term steady
SpotInterruptible
Dedicated HostCompliance

Steady 3-year workload → Reserved.

Spot interruption → 2-minute notice.

Section 6 – EC2 (Solutions Architect Level)

▼

Placement Groups

  • Cluster — Same AZ. Low latency. HPC workloads.
  • Spread — Max 7 instances per AZ. Reduces hardware failure risk.
  • Partition — Large distributed systems.

Exam trigger: Low latency → Cluster.

Elastic Network Interface (ENI)

  • Has private IP
  • Can attach/detach
  • Can have multiple IPs
  • Security groups attached to ENI

Exam trigger: Move network identity → Use ENI.

EC2 Hibernate

  • Saves RAM to EBS
  • Faster restart
  • Root volume must be EBS
  • Supported only on certain instance types

Section 7 – EC2 Instance Storage

▼

Amazon EBS

  • Block storage
  • AZ-scoped
  • Persist after stop
  • Network attached

EBS Volume Types

TypeUse
gp2/gp3General purpose
io1/io2High IOPS
st1Throughput HDD
sc1Cold HDD

Multi-Attach → io1/io2 only.

Snapshots

  • Stored in S3
  • Region-scoped
  • Incremental
  • Can create volume in any AZ in same region
  • Inherit encryption

Remember: EBS → AZ. Snapshot → Region.

EBS Encryption

  • Uses KMS
  • Snapshots inherit encryption
  • Can enable encryption by default

Delete-on-Termination

Root EBS deleted by default. Can modify setting.

AMI

Template for EC2. Includes OS + configuration. Can copy across Regions.

Exam trigger: Deploy identical setup in new region → Copy AMI.

Instance Store

  • Ephemeral storage
  • Lost on stop/terminate
  • Very high performance
  • Used for cache/temp data

Keyword: Lost on stop → Instance Store.

Amazon EFS

  • File storage (NFS)
  • Multi-AZ
  • Shared across instances
  • Automatically scales

Keyword: Shared storage → EFS.
